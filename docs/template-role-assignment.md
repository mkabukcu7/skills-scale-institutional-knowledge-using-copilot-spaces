## Role Interaction Matrix

Use this matrix to clarify primary interaction points and collaboration frequency:

| Role Pair | Interaction Frequency | Primary Collaboration Areas | Communication Channel |
|-----------|----------------------|----------------------------|----------------------|
| Product Owner ↔ Developers | Daily | Requirements clarification, sprint planning | Standup, Slack, PR comments |
| Product Owner ↔ QA Lead | Weekly | Acceptance criteria, testing priorities | Planning sessions, Reviews |
| Project Manager ↔ All Roles | Weekly | Status updates, blockers, dependencies | Status meetings, Project board |
| Business Analyst ↔ Product Owner | Daily/Weekly | Requirements refinement, backlog grooming | Workshops, Documentation |
| UX/UI Designer ↔ Developers | As needed | Design implementation, feasibility | Design reviews, PR comments |
| QA Lead ↔ Developers | Daily | Defect triage, test automation | Standup, Slack |
| Change Manager ↔ Project Manager | Weekly | Change impact assessment | Change board, Email |

## Escalation Path

1. **Team Level**: Team members → Technical Lead / Product Owner
2. **Project Level**: Technical Lead / Product Owner → Project Manager / Product Manager
3. **Program Level**: Project Manager / Product Manager → Executive Sponsor
4. **Executive Level**: Executive Sponsor → Senior Leadership

**Escalation Criteria**:
- Blocker unresolved for > 48 hours
- Risk impact > $X or > Y days delay
- Scope change requiring additional resources
- Quality issue affecting release
- Stakeholder conflict requiring mediation
