# Team Member Onboarding Checklist

## Purpose
This checklist ensures new team members receive consistent onboarding and have access to all necessary resources, tools, and context to contribute effectively to OctoAcme projects.

## Pre-Arrival Setup (Manager/Lead)

- [ ] Create necessary accounts (GitHub, project tools, communication platforms)
- [ ] Add to relevant teams and distribution lists
- [ ] Grant access to repositories and project boards
- [ ] Prepare workspace or development environment
- [ ] Assign onboarding buddy
- [ ] Schedule first-week meetings (1:1s, team intro, key stakeholders)

---

## Week 1: Orientation and Foundation

### Documentation and Context
- [ ] Review [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read [Roles and Personas](octoacme-roles-and-personas.md) document
- [ ] Understand your role's responsibilities and interactions
- [ ] Review current project charter or one-pager
- [ ] Access and review project roadmap
- [ ] Familiarize with project board and workflow

### Tools and Access
- [ ] Verify GitHub access and repository permissions
- [ ] Set up local development environment (if applicable)
- [ ] Configure communication tools (Slack, Teams, etc.)
- [ ] Access project management tools and dashboards
- [ ] Test build and deployment processes (if applicable)

### Team Connections
- [ ] Meet with onboarding buddy for introductions and Q&A
- [ ] Attend team standup or sync meeting
- [ ] 1:1 with manager/lead to discuss goals and expectations
- [ ] Introduction to key stakeholders and collaborators
- [ ] Attend sprint planning or team meeting

### Initial Tasks
- [ ] Review current sprint backlog
- [ ] Shadow team member on typical tasks
- [ ] Complete first small task or documentation contribution
- [ ] Ask questions and document answers for future reference

---

## Week 2-4: Integration and Contribution

### Process and Practices
- [ ] Review [Project Initiation](octoacme-project-initiation.md) guide
- [ ] Review [Project Planning](octoacme-project-planning.md) process
- [ ] Review [Execution and Tracking](octoacme-execution-and-tracking.md) workflows
- [ ] Understand [Risk Management & Communication](octoacme-risks-and-communication.md) practices
- [ ] Review [Release and Deployment](octoacme-release-and-deployment.md) procedures
- [ ] Read [Retrospective and Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md)

### Hands-On Work
- [ ] Complete first significant work item
- [ ] Participate in code/document review process
- [ ] Submit first pull request following team conventions
- [ ] Participate in team ceremonies (standup, planning, review, retro)
- [ ] Begin regular contributions to sprint backlog

### Relationship Building
- [ ] Schedule 1:1s with cross-functional partners
- [ ] Attend stakeholder meetings or demos
- [ ] Join relevant community of practice or special interest groups
- [ ] Provide feedback on onboarding experience

---

## 30-Day Check-in

- [ ] Review progress with manager/lead
- [ ] Discuss any blockers or additional support needed
- [ ] Clarify expectations and success criteria
- [ ] Identify areas for skill development or training
- [ ] Confirm understanding of team norms and processes

---

## 60-Day Check-in

- [ ] Demonstrate autonomy on standard tasks
- [ ] Contributing regularly to team objectives
- [ ] Participating actively in team ceremonies
- [ ] Building relationships with stakeholders
- [ ] Identifying improvement opportunities

---

## 90-Day Check-in

- [ ] Fully ramped and contributing at expected level
- [ ] Seeking feedback and continuous improvement
- [ ] Mentoring or supporting newer team members
- [ ] Contributing to process improvements
- [ ] Discuss long-term goals and career development

---

## Role-Specific Onboarding

### For Developers
- [ ] Set up development environment and toolchain
- [ ] Review coding standards and contribution guidelines
- [ ] Complete first code contribution and review cycle
- [ ] Understand CI/CD pipeline and deployment process
- [ ] Familiarize with testing strategy and requirements

### For Product Owner/Product Manager
- [ ] Review product vision and strategy documents
- [ ] Understand customer personas and use cases
- [ ] Access analytics and metrics dashboards
- [ ] Meet with key customers or stakeholders
- [ ] Review historical product decisions and rationale

### For QA Lead
- [ ] Review testing strategy and test plans
- [ ] Access test environments and tools
- [ ] Understand quality metrics and reporting
- [ ] Review defect management process
- [ ] Familiarize with automation framework

### For Business Analyst
- [ ] Review existing business requirements documentation
- [ ] Understand business domain and workflows
- [ ] Access to business stakeholders
- [ ] Review requirement templates and standards
- [ ] Familiarize with modeling tools

### For Project Manager
- [ ] Review active project portfolio
- [ ] Understand reporting and status update cadence
- [ ] Access project management tools and templates
- [ ] Review risk registers and issue logs
- [ ] Meet with project sponsors and stakeholders

### For UX/UI Designer
- [ ] Review design system and style guides
- [ ] Access design tools and repositories
- [ ] Understand user research repository
- [ ] Review accessibility standards
- [ ] Meet with key users or customers

### For Change Manager
- [ ] Review change management process and policies
- [ ] Access change log and historical changes
- [ ] Understand approval workflows
- [ ] Familiarize with impact assessment framework
- [ ] Review communication templates
